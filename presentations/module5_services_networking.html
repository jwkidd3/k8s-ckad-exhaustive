🔌 Services

The Problem They Solve

  • Pods are ephemeral — IPs change on restart
  • Services give a stable virtual IP that load-balances to current Pods
  • Selector matches Pod labels → endpoints

Service Types

TypeReachable from
ClusterIPinside the cluster
NodePortnode IP : 30000-32767
LoadBalancercloud LB (external)
ExternalNameDNS CNAME to external host

ClusterIP Manifest

apiVersion: v1
kind: Service
metadata:
  name: myapp
spec:
  type: ClusterIP
  selector:
    app: myapp
  ports:
  - protocol: TCP
    port: 80         # Service port
    targetPort: 80   # Pod port

kubectl expose deployment myapp --port=80 --target-port=80 generates the manifest above.

🔧 Troubleshooting Services

If a Service Doesn't Work, Check…

  1. kubectl get endpoints <svc> — empty means selector didn't match
  2. Service selector labels vs Pod labels
  3. targetPort matches the actual container port
  4. protocol (TCP vs UDP)
  5. NetworkPolicy isn't blocking traffic
  6. Pod is in Running state

Useful Probes from Inside

# Throwaway pod for testing
kubectl run probe --rm -it --restart=Never \
  --image=alpine/curl:3.14 -- sh

# Inside:
curl http://myapp:80
nslookup myapp
curl http://myapp.default.svc.cluster.local

Most common bug: the Service selector says run: myapp but the Deployment template says app: myapp. kubectl describe svc shows the empty endpoints.

🚪 Ingress

Ingress vs Service

  • Service load-balances at L4
  • Ingress routes at L7 — by host and path
  • Implemented by an Ingress Controller (e.g. ingress-nginx) — you must install one
  • Same Ingress resource works across controllers

Current Spec

  • apiVersion: networking.k8s.io/v1
  • pathType: Prefix | Exact | ImplementationSpecific — required
  • backend.service.name + backend.service.port.number (not the old serviceName/servicePort)
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: hello-world-ingress
spec:
  rules:
  - host: hello-world.exposed
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: web
            port:
              number: 3000

🚧 NetworkPolicy

Default Posture

  • Without any NetworkPolicy, all Pods can talk to all Pods
  • A NetworkPolicy selects Pods and defines what's allowed
  • The moment a Pod is selected by any policy, it becomes "isolated" — only explicitly allowed traffic gets through
  • Requires a network plugin that supports NetworkPolicy (Calico, Cilium, etc.)

Anatomy

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-from-consumer
  namespace: k2
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes: [Ingress]
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          role: consumer
    ports:
    - protocol: TCP
      port: 80

🎯 CKAD Exam Tips

Before the Exam

  • Memorize the kubectl run --dry-run=client -o yaml stub
  • Set up your kubectl aliases: k, kn, kgp
  • Bookmark these sections of kubernetes.io: Pods, Deployments, Services, Ingress, RBAC
  • Know kubectl explain <kind>.<field> — your offline schema reference

During the Exam

  • Flag hard questions and come back — never burn 15 minutes on one
  • Use --namespace in every command unless default is set
  • Read all the requirements in a question before starting
  • Verify with kubectl get after every change
  • Time check at the 60-minute mark

Passing score is 66%. You don't need perfection — you need accuracy on the questions you do attempt.
