☸️ What is Kubernetes?

An Orchestrator for Containers

  • Schedules containers onto worker nodes
  • Heals failed workloads automatically
  • Scales applications horizontally
  • Exposes them through stable network endpoints
  • Configures them through ConfigMaps and Secrets

Declarative Model

You describe the desired state in YAML. The control plane works continuously to make reality match that state.

Cluster Anatomy

┌─────────────────────────────┐ │ Control Plane │ │ api-server scheduler │ │ controller-mgr etcd │ └──────────────┬──────────────┘ │ ┌───────────┼───────────┐ ▼ ▼ ▼ worker1 worker2 worker3 (kubelet + container runtime)

Your kind cluster runs this whole topology inside Docker containers on a single Cloud9 host.

🛠️ Delivery Environment: Cloud9 + kind

AWS Cloud9

  • Browser-based IDE running on Amazon Linux
  • Docker is pre-installed
  • One Cloud9 instance per student
  • x86_64 Linux — all install commands target this arch

kind — Kubernetes IN Docker

  • Each Kubernetes node is a Docker container
  • Whole cluster lives on one host — perfect for labs
  • Fast to create, fast to throw away
  • We use a 4-node config: 1 control-plane + 3 workers

Why kind for CKAD prep?

The CKAD exam itself runs against multi-node clusters. kind reproduces that topology locally so scheduling, NodePort behavior, and taints all behave realistically.

⌨️ Install kubectl + kind

1. Confirm Docker

$ docker info | head -5

If Docker isn't running: sudo service docker start

2. Install kubectl

curl -LO "https://dl.k8s.io/release/$(curl -L -s \
  https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 \
  kubectl /usr/local/bin/kubectl
kubectl version --client

3. Install kind v0.27.0

curl -Lo ./kind \
  https://kind.sigs.k8s.io/dl/v0.27.0/kind-linux-amd64
chmod +x ./kind
sudo mv ./kind /usr/local/bin/kind
kind version

Version note: kind v0.27 ships kindest/node v1.32, which is current for CKAD as of 2026.

🚀 Create & Verify the Cluster

Create from kind-config.yaml

$ cd ~/environment/k8s-ckad-exhaustive
$ kind create cluster --config kind-config.yaml
Creating cluster "cluster" ...
 ✓ Ensuring node image (kindest/node:v1.32.0)
 ✓ Preparing nodes 📦 📦 📦 📦
 ✓ Writing configuration 📜
 ✓ Starting control-plane 🕹️
 ✓ Installing CNI 🔌
 ✓ Joining worker nodes 🚜

Verify Nodes

$ kubectl get nodes
NAME                    STATUS   ROLES
cluster-control-plane   Ready    control-plane
cluster-worker          Ready    <none>
cluster-worker2         Ready    <none>
cluster-worker3         Ready    <none>

Install metrics-server

$ kubectl apply -f components.yaml
$ kubectl -n kube-system rollout status \
    deployment/metrics-server

The --kubelet-insecure-tls flag is already baked into components.yaml so it works on kind out of the box.

🚀 Let's Get Your Cluster Online!

🧹 Tearing Down + Re-creating

Delete a Cluster

$ kind delete cluster --name cluster
Deleting cluster "cluster" ...

Some labs benefit from a clean slate — feel free to delete and recreate. It takes about 60 seconds.

Recover a Broken Cluster

  • If a node is stuck NotReady for >5 min, delete and recreate.
  • If kubectl hangs, check Docker: docker ps
  • If a Pod ends up Pending after a fresh start, check that metrics-server is running before re-running the lab.
